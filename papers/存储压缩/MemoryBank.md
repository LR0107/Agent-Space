# MemoryBank: Enhancing Large Language Models with Long-Term Memory

**AAAI 2024**

## 一.研究背景

目前的 ChatGPT、GPT-4 等大语言模型虽然很强，但有一个明显缺陷：

> 它们只能利用当前上下文窗口的信息，无法真正长期记住用户。

作者认为：真正的长期交互AI必须拥有类似人类的长期记忆能力。

---

## 二.研究方法

MemoryBank  
│  
├── Memory Storage  
│ （记忆存储）  
│  
├── Memory Retrieval  
│ （记忆检索）  
│  
└── Memory Updating  
（记忆更新）

流程：

用户对话 → 存入记忆库→提取重要事件→形成用户画像

新问题来时 → 检索相关记忆 → 一起输入LLM → 生成回答

---

### 1.记忆存储（Memory Storage）

设计了三层记忆。

**第一层：原始对话记录**

保存所有历史聊天。

按时间顺序存储。

**第二层：事件摘要（Event Summary）**

每天结束后让LLM自动总结。

这样可以压缩信息量。

**第三层：用户画像（User Portrait）**

让LLM分析：

```
用户性格是什么？
用户兴趣是什么？
```

形成长期人格画像。

---

### **2.记忆检索（Memory Retrieval）**

存了几千条记忆后，怎么找到有用的信息？

作者借鉴了：**Dense Retrieval（稠密检索）**

流程：

当前问题
     ↓
编码成向量

历史记忆
     ↓
编码成向量

计算相似度
     ↓
找最相关记忆

例如：

用户问：你还记得我喜欢什么相机吗？

系统会从记忆库中找到：用户买了索尼A7M4

然后送给LLM。

实现：FAISS向量数据库——用于快速搜索历史记忆。

---

### 3.记忆更新（Memory Updating）

作者借鉴了心理学中的：**艾宾浩斯遗忘曲线**

人的记忆规律：

刚学完
 ↓
忘得很快
 ↓
之后逐渐稳定

其中：

- R：记忆保留率
- t：时间
- S：记忆强度

**记忆强化机制**

当某段记忆再次被提及时：

```
S = S + 1
t = 0
```

即：

- 记忆强度增加
- 遗忘速度减慢

### 工作流程

**1.存储记忆（Memory Storage）**

用户每次与 AI 聊天后：

- 保存原始对话记录
- 按日期整理
- 记录时间戳

**2.生成事件摘要和用户画像**

系统定期利用 LLM 对历史对话进行总结：事件摘要和用户画像

这样可以减少存储冗余，同时提取关键信息。

**3.检索相关记忆（Memory Retrieval）**

当用户提出新问题时：

系统会：

- 将当前问题编码成向量
- 在记忆库中搜索最相似的历史记忆（计算相似度）
- 找到相关记录

**4.构造增强提示词（Memory-Augmented Prompt）**

把以下内容一起发送给大模型：

```
当前问题+相关历史记忆+用户画像+事件摘要
```

这样模型就能生成个性化回答。

**5.生成回复**

LLM 根据：

- 当前对话
- 检索到的记忆
- 用户画像

生成回答

**6.记忆更新（Memory Updating）**

回复结束后，MemoryBank 会更新记忆状态：

如果记忆被频繁提到，记忆强度增加。如果长期未使用，如果长期未使用。

## 三.实验设计

### 实验设置

**数据构造**

作者让 ChatGPT 扮演：15个虚拟用户

每个人：

- 不同性格
- 不同兴趣

持续聊天：10天、共覆盖：450个话题

然后建立记忆库。

**测试问题**

人工设计：194个记忆测试问题

模型需要从历史记录中找到答案。

**评价指标**

1.Retrieval Accuracy：检索是否找到正确记忆

2.Response Correctness：答案是否正确

3.Contextual Coherence：回答是否自然连贯

4.Ranking Score：人工综合排名

### Q&A

**1.MemoryBank和RAG有什么区别？**

MemoryBank本质上也是一种基于检索增强生成（RAG）的框架，因为它同样通过向量检索获取历史信息，再将检索结果拼接到Prompt中供LLM生成回答。

但与传统RAG不同的是，MemoryBank不仅存储知识，还维护了用户画像和事件摘要，并引入了遗忘机制来动态管理记忆。因此它更关注长期个性化交互，而不是知识问答。

**2.这篇论文最大的创新点是什么？**

最大的创新点是引入了受艾宾浩斯遗忘曲线启发的动态记忆更新机制。

以前的大部分长期记忆系统主要解决“如何存储和检索记忆”，而MemoryBank进一步考虑“如何遗忘记忆”。

系统会根据时间和记忆被调用的频率动态调整记忆强度，使重要记忆长期保留，而不重要的信息逐渐淡化，更接近人类记忆机制。

**3.为什么要引入遗忘机制？**

如果所有对话永久保存，会导致两个问题：

第一是存储规模持续增长；

第二是检索噪声增加。

随着历史对话越来越多，很多无关信息会干扰记忆检索。
