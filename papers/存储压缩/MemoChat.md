# MemoChat: Tuning LLMs to Use Memos for Consistent Long-Range Open-Domain Conversation

## 一、研究背景

当前 ChatGPT、GPT-4 等大模型虽然对话能力很强，但存在一个明显问题：对话一长，就容易“忘事”。

很多模型会：

- 回答错误
- 回忆不完整
- 直接忘记

论文把这种场景称为：Long-Range Open-Domain Conversation（长程开放域对话）

### 现有解决方案的问题

**方法1：扩大上下文窗口**

例如：

- Claude 100K
- GPT-4 Turbo 128K
- LongNet

思路：把历史聊天记录全部塞进去。

问题：

- 成本高
- 速度慢
- 窗口再大也不是无限
- 中间内容容易被忽略（Lost in the Middle）

**方法2：外部记忆库**

例如：

- MemoryBank
- MPC

思路：

- 把历史内容存进数据库
- 用向量检索找回

问题：

- 检索错误会累计
- 依赖额外模块
- 系统复杂

作者提出：不如让模型自己学会做笔记。

就像人一样：

聊天时：

- 记录重要信息
- 分类整理
- 以后需要时再翻笔记

## 二、研究方法

Memorization → Retrieval → Response

先整理历史对话。根据当前问题寻找相关记忆。结合检索到的记忆生成回答。

```
聊天历史
      ↓
生成 Memo
      ↓
保存 Topic + Summary
      ↓
用户新问题
      ↓
检索相关 Memo
      ↓
生成回答
```

Memo 会被保存成：

| Topic              | Summary  |
| ------------------ | -------- |
| Quantum Physics    | 讨论量子叠加   |
| Business Etiquette | 商务礼仪注意事项 |

当用户问：“你还记得量子纠缠吗？”

系统会先定位：Quantum Physics

然后找到对应记录。而不是搜索全部聊天记录。

用指令微调（Instruction Tuning）教模型学会使用 Memo。

**任务1：Memo Writing**

让模型学会：把对话整理成笔记。

输入：

```
用户：我担心妹妹两周没联系我...
用户：我要去火车站
```

输出：

```
[ 
{ "topic":"担心妹妹", "summary":"用户担心妹妹失联"}, 
{ "topic":"打车", "summary":"用户前往火车站"}
]
```

**任务2：Memo Retrieval**

让模型学会：根据问题找到相关 Memo。

例如：

问题：帮我准备办公室聚会

候选主题：

```
1. 机场送别
2. 聚会准备
```

输出：2

表示：当前问题与“聚会准备”相关。

**任务3：Chat with Memo**

让模型学会：利用检索出的 Memo 回答问题。

输入：

- 当前问题
- 检索结果
- 最近对话

输出：

- 最终回复

### 训练数据怎么来的？

作者没有人工构造大量数据。而是重构已有公开数据集。

TopiOCQA：多主题长对话数据集

用于：

- Topic识别
- 对话切分

DialogSum：对话摘要数据集

用于：Summary生成

Alpaca-GPT4：高质量指令数据

用于：训练聊天能力

最终构造：

| 任务             | 数据量  |
| -------------- | ---- |
| Memo Writing   | 3046 |
| Memo Retrieval | 3654 |
| Chat with Memo | 3300 |

总计约：1万条训练样本。

这样训练完以后，模型自己就学会：

- 怎么总结
- 怎么分类
- 怎么检索
- 怎么利用记忆回答

## 三、实验设计

**实验目标**

验证：MemoChat 是否真的能让模型记住长期对话内容？

测试模型

作者测试了：

API模型：ChatGPT

开源模型

- FastChat-T5-3B
- Vicuna-7B
- Vicuna-13B
- Vicuna-33B

**自建测试集 MT-Bench+**

作者觉得原有测试集不够考察长期记忆。于是自己扩展了 MT-Bench。

构造三种问题：

① Retrospection（回忆）

测试：还能不能记住前面聊过的内容？

例如：刚才我们说了几种机器学习？

 ② Continuation（延续）

测试：能否接着以前的话题继续讨论？

例如：根据之前那个故事继续写下去。

③ Conjunction（跨主题关联）

测试：能否把不同话题联系起来？

例如：反垄断法会如何影响机器学习行业？

需要同时回忆：机器学习、反垄断法两个主题。

### 实验结果

**结果1：MemoChat 显著提升长期记忆**

平均得分（GPT4评审）：

| 模型               | 平均分   |
| ---------------- | ----- |
| ChatGPT-2k       | 51.89 |
| MPC              | 54.52 |
| MemoryBank       | 42.44 |
| MemoChat-ChatGPT | 70.76 |

提升幅度：51.89 → 70.76

提高约：36%

**结果2：开源模型也有效**

Vicuna-33B：

| 模型                  | 平均分   |
| ------------------- | ----- |
| 原版Vicuna            | 较低    |
| MemoChat-Vicuna-33B | 61.52 |

说明：不是 ChatGPT 特例。MemoChat 具有较好的泛化性。

如果是 GPT-4：甚至不用训练，作者做过实验直接 Prompt

开源模型（Vicuna、Llama）：需要真正微调。

## 四、总结

MemoChat本质上是一种“训练模型使用记忆”的微调方案

### 创新点

把长期对话拆成「记忆-检索-回答」三阶段，把记忆管理能力训练进模型

### 不足

本质上还是摘要记忆，会产生信息损失

检索能力不稳定，作者的 Retrieval 本质是LLM分类，稳定性比向量检索差。

没有真正解决无限记忆，Memo也会越来越长。

### Q&A

**1.MemoChat 为什么要使用 Topic 和 Summary？**

Topic 和 Summary 是一种结构化记忆表示。Topic 用于快速定位相关内容，相当于索引；Summary 用于保存关键信息，相当于压缩后的知识表示。相比直接存储全部对话，这种结构能够降低检索成本。

**2.MemoChat 为什么后来没有成为主流？**

因为RAG的发展速度更快。向量检索能够直接访问原始信息，不会像摘要那样丢失细节，而且检索准确率更高。因此后续很多工作采用了“Memory + RAG”的混合方案，而不是单独使用MemoChat。MemoChat更多被看作长期记忆Agent的早期探索。

**3.实验设计有没有问题？**

有一定局限性。作者主要使用GPT-4作为评审模型进行自动评分，而不是大规模人工评价，可能存在偏差。此外，测试集规模较小，只有54个长期记忆问题。
