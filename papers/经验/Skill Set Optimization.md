# Skill Set Optimization: Reinforcing Language Model Behavior via Transferable Skills

问题：如何让大语言模型智能体在不修改模型参数、不进行强化学习微调的情况下，利用环境中的奖励反馈不断改进自己的行为，并把已经学会的经验迁移到新任务中？

核心做法是：体过去的交互轨迹中提取高奖励、可复用的行为片段，将其概括成自然语言“技能”，在之后决策时放入提示词中；同时持续删除那些实际效果不好的技能。

**ICML 2024**

## 一、研究背景

大语言模型不仅可以回答问题，也可以作为“智能体”与环境交互

这类任务具有明显的**序列决策**特征：模型必须按正确顺序完成多个动作。

交互环境通常会给模型奖励

传统强化学习可以根据奖励更新策略参数，但对于现代大语言模型来说存在几个问题：

- 微调成本很高；
- 需要大量交互数据；
- 长任务中的“信用分配”困难，即很难判断最终成功是由哪一步造成的；
- 对于 GPT-4 一类闭源模型，研究者无法直接更新其参数。

因此，作者不考虑修改模型权重，而是研究一种 ICL，即上下文学习：

> 不训练模型，只改变输入给模型的提示内容，从而改善其后续行为。

### 现有“记忆”方法存在不足

已有方法会让智能体保存：

- 过去的成功轨迹；
- 对失败原因的反思；
- 关于环境规则的经验；
- 已经学会的技能。

但作者认为这些方法仍有几个缺陷：

1. **缺少持续评估**  
   一条记忆一旦保存，通常不会根据后续表现判断它是否真的有用。
2. **缺少明确目标**  
   很多记忆只是一般性的经验，而不是“要达到什么状态、具体怎么做”。
3. **检索机制不够明确**  
   当前状态下应该调用哪条记忆，不一定容易判断。
4. **成功经验过长**  
   直接把完整成功轨迹放入上下文，会消耗大量上下文长度，而且不容易迁移到新任务。

## 二、研究方法

论文中的一个技能由三部分组成：

1. **初始状态**  
   用来判断该技能适合在什么情况下使用。
2. **目标状态**  
   表示执行技能后希望环境达到的结果。
3. **动作指令序列**  
   告诉模型应当如何完成这个子目标。

### SSO 循环

环境交互→构建技能→筛选技能→下一轮交互

**1.与环境交互并收集轨迹**

首先，LLM智能体在当前任务中执行一次完整尝试。

模型每一步都会看到：

- 当前任务；
- 当前环境状态；
- 从技能集中检索出的相关技能；
- 可以执行的动作集合。

然后，模型输出：

- 当前准备执行哪个技能；
- 下一步动作是什么。

**2. 技能集构建**

技能构建包括四个步骤：

**2.1 提取子轨迹：Extract**

作者不会直接使用整条轨迹，而是从中提取较短的连续片段，即“子轨迹”。

**2.2 为候选技能评分：Score**

每一对子轨迹会根据三类指标评分。

**相似度**

对于**最新轨迹中的每一条子轨迹**，SSO会去历史上的若干条完整轨迹中寻找：与它长度相同、最相似的一条历史子轨迹。是为了找到在不同任务尝试中反复出现的行为模式。

这两条片段会组成一个候选子轨迹对。

比较两段子轨迹中的：

- 状态语义相似度；
- 动作语义相似度。

论文使用文本嵌入和余弦相似度来计算。

相似度越高，越可能表示同一个可复用技能。

**奖励**

先真实运行完整任务→得到状态、动作、奖励轨迹→事后切出子轨迹→用轨迹中已有奖励给子轨迹评分

考察执行子轨迹之后获得的未来奖励：

不仅考虑当前动作的奖励，也考虑该行为是否推动了后续任务成功。

**长度**

较长的多步片段通常包含更完整的行为模式，因此也被纳入评分，但权重较小。

论文采用的评分权重为：

- 相似度：1.0；
- 奖励：0.1；
- 长度：0.01。

**2.3 采样候选技能：Sample**

候选子轨迹之间可能高度重叠。

如果都生成技能，会产生大量重复内容。

因此，作者使用 **束搜索（beam search）**，选择总评分较高、但彼此不重叠的一组子轨迹。

**2.4 生成自然语言技能：Generate**

选出子轨迹后，作者让大语言模型将多条相似子轨迹概括为：

- 一个通用子目标；
- 一组带编号的指令。

例如，两条轨迹分别描述：

- 将巧克力放到炉子上并启动炉子；
- 将冰放到炉子上并启动炉子。

模型可能生成：

> **目标：待加热物质已经放在开启的炉子上。**

> 1. 找到待加热物质；
> 2. 将物质移动到炉子上；
> 3. 启动炉子。

**LLM只用于最后的语言概括步骤。子轨迹提取、相似度计算和评分主要由算法完成。**作者认为，让LLM直接负责所有筛选和评分会更不稳定，也更昂贵。

“Extract”阶段允许大量重叠，目的是保证不漏掉潜在技能；

“Sample”阶段再消除重叠，选择总评分较高的一组。

**3. 技能检索**

技能集可能包含很多技能，但每一步不会全部放入上下文。

SSO将当前状态与每个技能的初始状态计算语义相似度，选出最相关的技能。

实验中，每一步最多检索 **3个技能**。

例如，当模型看到：

> 房间里有炉子、冰块和温度计。

系统可能检索出：

- 将物质移动到炉子并加热；
- 使用温度计读取温度；
- 聚焦实验对象。

这样可以减少无关信息和提示词长度。

**4. 技能集优化与剪枝：Refinement**

这是SSO与很多已有记忆方法的重要区别。

模型在行动时需要自我报告：

> 我现在正在执行哪个子目标或技能？

如果模型声明正在执行某个技能，系统就记录从该时刻开始的未来奖励。

一个技能如果长期不能产生正向奖励，就会被剪枝。

这使技能集不是静态知识库，而是一个会不断变化的“经验库”。

---

### 工作流程

- **环境交互**  
  LLM 智能体根据当前状态和已有技能选择动作，完成一次任务尝试，并记录状态、动作和奖励轨迹。
- **提取候选技能**  
  从历史轨迹中截取长度为 2～5 步的子轨迹，并寻找不同轨迹中相似的行为片段。
- **候选技能评分**  
  根据三项指标评分：
  - 状态和动作是否相似；
  - 后续获得的奖励是否高；
  - 行为片段是否包含较完整的多步操作。
- **生成技能**  
  选择高分且不重复的子轨迹，让 LLM 将其概括成：
  - 一个子目标；
  - 一组完成该子目标的操作指令。
- **检索并使用技能**  
  在下一次决策时，将当前状态与技能的适用状态进行匹配，最多选取 3 个相关技能加入提示词，帮助 LLM 决策。
- **技能优化与淘汰**  
  记录技能被使用后带来的未来奖励。若某个技能长期不能产生正向收益，就将其删除。

## 三、实验设计

### 实验设置

**数据集**

- **ScienceWorld**：文本形式的科学实验环境；**18类任务**，每类包含约 **7到10个测试变体**
- **NetHack / MiniHack**：带有低层字符动作的网格游戏环境。

**ScienceWorld实验**

**Baseline**

| 方法        | 核心思路             |
| --------- | ---------------- |
| ReAct     | 让模型在行动前进行推理和规划   |
| Reflexion | 失败后生成反思，再次尝试     |
| CLIN      | 从过去经验中总结环境动作关系   |
| SSO       | 从高奖励子轨迹构建并持续筛选技能 |

使用 GPT-4。

### 实验结果

| 方法        | 平均得分     |
| --------- | -------- |
| ReAct     | 29.6     |
| Reflexion | 39.4     |
| CLIN      | 62.2     |
| **SSO**   | **83.7** |

迁移模式：从一组任务中学习到的技能，能否迁移到同类型的新任务？

| 方法      | 平均得分     |
| ------- | -------- |
| CLIN    | 52.7     |
| **SSO** | **60.1** |

NetHack：相比基线提升约 **40%**

**技能与完整轨迹的比较**

作者还比较了三种上下文：

- 不提供额外信息；
- 提供少量完整轨迹；
- 提供SSO生成的技能。

结果表明，技能通常优于完整轨迹，特别是在NetHack中。

一个重要原因是：完整轨迹的上下文长度平均约为技能上下文的5倍。

因此，SSO不仅表现更好，也更节省上下文空间。

**消融实验**

作者分别移除SSO的三个关键组成部分：

1. **不进行技能剪枝**；
2. **评分时不使用奖励**；
3. **不使用相似子轨迹提取**，而是简单选择奖励前的若干步。

结果显示，移除任何一个部分都会导致性能下降。

## 四、总结

### 创新点

技能不是永久保存，而是持续用奖励检验和剪枝

同时支持“快速适应”和“跨任务迁移”：既可以帮助模型在同一个新任务上多次尝试、逐渐改进，也能把从训练任务中学到的技能迁移到未见过的任务变体。

### 不足

对负面经验利用不足：它不会详细总结“为什么失败”或“什么动作应该避免”。

技能价值归因并不精确：模型需要自己报告“当前正在执行哪个技能”，然后系统把后续奖励归给该技能。

可能出现：

- 模型其实没有执行某技能，却说执行了；
- 多个技能共同导致成功，但奖励只归给其中一个；
- 后续奖励来自其他行为，却被错误归因给当前技能。

### Q&A

**1.SSO中的“技能”具体是什么？**

一个技能主要包含三个部分：**初始状态、目标状态和操作指令**。初始状态用于判断当前环境是否适合使用该技能；目标状态表示技能完成后应达到的目标；操作指令则说明如何实现该目标。

**2.评分中的相似度是和谁比较？**

相似度比较的是：最新轨迹中的某一条子轨迹，与历史完整轨迹中长度相同的最相似子轨迹。系统分别计算状态文本和动作文本的嵌入余弦相似度，再进行平均。它不是比较同一条轨迹内部的两个动作，也不是直接将子轨迹与技能库比较。

**3.候选子轨迹大量重叠，论文如何处理？**

提取阶段允许子轨迹重叠，以提高召回率，避免遗漏潜在技能。之后在采样阶段，作者使用带约束的束搜索，要求最终选择的子轨迹不能相互重叠。生成技能后，LLM还会进一步删除语义重复的技能。

**4.SSO如何判断一个技能是否真正有用？**

模型在行动时需要报告自己当前正在执行哪个技能。系统从该时刻开始计算之后获得的累计奖励，并将其加入该技能的观测价值。如果技能的观测价值低于或等于设定阈值，实验中阈值为0，该技能就会被删除。也就是说，技能是否保留最终由环境奖励决定。

**5.实验中的适应和迁移有什么区别？**

适应实验允许智能体在同一个新任务变体上重复尝试，并在不同尝试之间更新技能集，主要考察快速学习能力。迁移实验则先在若干训练任务变体上学习技能，再将技能集用于未见过的测试变体，主要考察技能能否跨任务泛化。
