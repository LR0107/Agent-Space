# MemP: Exploring Agent Procedural Memory

**ACL 2026**

## 一、研究背景

现代 LLM 智能体已经可以调用工具、浏览网页、操作软件界面、制定旅行计划，甚至完成较长的多步骤任务。

但它们有一个明显问题：

- 每遇到一个新任务，往往重新探索；
- 即使之前做过相似任务，也未必会复用经验；

- 环境稍有变化，原有提示模板可能失效；
- 长任务会消耗大量时间和 Token。

程序性记忆可以理解为：

> **关于“怎么做”的记忆，而不是关于“是什么”的知识。**

在人类认知中，骑自行车、打字、系鞋带都属于程序性记忆。人不需要每次重新推理动作顺序，而是可以熟练执行。

对应到智能体中，程序性记忆可以包括：

- 完成任务的步骤顺序；
- 应该调用哪些工具；
- 哪些操作容易失败；
- 遇到错误时如何恢复；

论文认为，现有智能体的程序性知识通常存在两个问题：

1. 被人工写在固定提示词或工作流里，比较脆弱；
2. 隐含在模型参数中，难以低成本修改和更新。

## 二、研究方法

过去的任务经验→构建记忆→检索相关记忆→执行新任务→根据结果更新记录

### 1. Build：构建程序性记忆

智能体完成任务后，会得到一条完整的交互轨迹，例如：

- 当前环境状态；
- 智能体采取的动作；
- 环境返回的观察结果；
- 最终是否成功；
- 一共使用了多少步骤。

作者研究了三种主要的记忆表达形式。

**Trajectory：完整轨迹**

直接保存成功任务的完整执行过程。

例如：

> 前往冰箱 → 拿鸡蛋 → 前往微波炉 → 放入鸡蛋 → 加热 → 取出 → 放入垃圾桶。

优点是信息具体，适合处理与历史任务非常相似的新任务。

缺点是轨迹较长，可能包含不必要的细节，占用上下文空间。

**Script：抽象脚本**

由大模型将完整轨迹总结成较高层的操作规则。

例如：

> 对于“加热某物并放入指定位置”的任务，先定位物品，再选择正确加热设备

优点是简洁、概括性强，更容易迁移到不同但结构相似的任务。

缺点是可能丢失具体操作细节。

**Proceduralization：轨迹与脚本结合**

同时提供：

- 具体的成功轨迹；
- 从轨迹中总结出的抽象操作规则。

这种方式兼顾“具体案例”和“一般方法”，也是实验中整体表现最好的构建策略。

### 2. Retrieve：检索相关记忆

智能体面对新任务时，不会把整个记忆库全部放进提示词，而是从中找出最相关的几条记忆。

论文采用向量表示和余弦相似度进行检索，即比较：

- 当前任务的向量；
- 历史任务或关键词的向量。

相似度越高，对应的程序性记忆越可能被取出。

作者比较了三种检索方式：

| 检索策略          | 做法                      | 特点          |
| ------------- | ----------------------- | ----------- |
| Random Sample | 随机选择记忆                  | 简单，但相关性不能保证 |
| Query         | 直接用完整任务描述作为检索键          | 能利用任务整体语义   |
| AveFact       | 从任务中提取关键词或关键事实，再计算平均相似度 | 更聚焦于任务核心要素  |

基于语义的 Query 和 AveFact 通常优于随机检索；AveFact 整体表现较强，说明仅比较完整句子还不够

### 3. Update：更新程序性记忆

MemP 的一个重要特点是：记忆不是写入后永久不变，而是根据后续任务结果不断调整。

作者研究了三种更新策略。

**Vanilla：直接追加**

每完成一批任务，就把新轨迹转化为记忆并加入记忆库。

问题是失败、噪声或重复经验也可能被加入，导致记忆库质量下降。

**Validation：成功验证**

只保留最终成功的轨迹，将失败轨迹和明显噪声丢弃。

这种方法提高了记忆质量，但没有主动修复已有的错误经验。

**Adjustment：反思与修正**

当智能体使用某条程序性记忆后仍然执行失败，就将以下信息结合起来：

- 原有记忆；
- 本次失败轨迹；
- 失败反馈。

然后让模型分析错误，并直接修改原有记忆。

Adjustment 的效果最好。

---

作者将智能体执行任务的过程表示为一个马尔可夫决策过程。

从：

> “根据眼前情况临时决定下一步”

变成：

> “结合眼前情况和以前成功做法来决定下一步”。

记忆更新则包含三类基本操作：添加新经验、删除过时或错误经验、修正已有经验。

- **构建记忆**：智能体完成历史任务后，从成功轨迹中提取两类信息：具体执行步骤和抽象操作规则，并存入程序性记忆库。
- **检索记忆**：遇到新任务时，根据任务语义或关键词，从记忆库中找出最相似、最相关的经验。
- **辅助执行**：将检索到的记忆加入上下文，指导智能体规划动作、调用工具，减少重复探索和错误操作。
- **动态更新**：任务结束后，根据成功或失败反馈，新增有效经验、删除无效记忆，或修正导致失败的旧记忆。

## 三、实验设计

**数据集**

TravelPlanner：复杂旅行规划任务。

ALFWorld：这是一个文字形式的家庭环境交互任务。

模型

实验使用了三类基础模型：

- **GPT-4o**；
- **Claude-3.5-Sonnet**；
- **Qwen2.5-72B-Instruct**。

**实验一：比较不同记忆构建方式**

对照组包括：

- No Memory；
- Script；
- Trajectory；
- Proceduralization。

目的是回答：

> 程序性记忆是否有效？哪种记忆形式更好？

| 方法                | Dev 成功率   | Test 成功率  | 平均步骤      |
| ----------------- | --------- | --------- | --------- |
| No Memory         | 39.28     | 42.14     | 23.76     |
| Script            | 66.67     | 56.43     | 18.52     |
| Trajectory        | 67.17     | 74.29     | 16.49     |
| Proceduralization | **87.14** | **77.86** | **15.01** |

结果说明：

- 所有记忆方式都明显优于无记忆；

- 两者结合时效果最好。

**实验二：比较不同检索方式**

对照：

- 随机检索；
- 完整 Query 检索；
- AveFact 关键词检索。

目的是判断：

> 记忆库中有正确经验还不够，智能体能否准确找到它？

结果显示，语义检索普遍优于随机选择

**实验三：比较不同更新机制**

对照：

- Vanilla；
- Validation；
- Adjustment。

实验把测试任务分成若干组。每执行完一组，更新一次记忆，再观察后续任务的表现。

结果表明，三种更新方式都有一定收益，但Adjustment最有效。

**实验四：与其他智能体方法比较**

作者在 ALFWorld 上比较：

- ReAct；
- ExpeL；
- AWM；
- MemP。

MemP 在开发集和测试集成功率上最高，同时平均步骤最少

## 四、总结

### 创新点

把程序性记忆从一个附加 Prompt，提升为需要持续优化的独立模块

程序性记忆可以跨模型迁移：将 GPT-4o 构建的记忆提供给 Qwen2.5-14B 后，较小模型的任务表现提高，同时执行步骤减少。

这个结果有实际意义：

> 可以让昂贵的强模型负责总结经验，让成本更低的小模型复用这些经验执行任务。

### 不足

检索机制比较简单：主要使用向量相似度检索

高度依赖明确的成功奖励：在 ALFWorld 中，环境直接返回任务是否成功；在 TravelPlanner 中，也有标准答案。但现实任务往往没有明确的 0/1 奖励

没有完整核算记忆系统本身的成本：论文主要展示执行阶段的步骤和 Token 减少，但构建与更新记忆也需要额外调用大模型

### Q&A

**1.这篇论文主要解决什么问题？**

它解决的是大模型智能体**不会稳定复用历史任务经验**的问题。传统智能体即使做过相似任务，下一次仍可能从头探索，造成步骤多、Token 消耗高、错误重复。MemP 将历史任务轨迹整理成独立的程序性记忆，使智能体在执行新任务时能够检索并复用过去的操作方法。

**2.什么是“程序性记忆”？它和普通知识记忆有什么区别？**

普通知识记忆主要回答“是什么”，例如“微波炉是一种加热设备”；程序性记忆回答“怎么做”，例如“先找到鸡蛋，把鸡蛋放进微波炉加热”。因此，程序性记忆保存的是操作步骤、工具调用顺序和错误恢复方法，更适合多步骤智能体任务。

**3.实验使用了哪些评价指标？**

在 ALFWorld 中，主要评价任务是否完成以及完成任务需要多少步骤；在 TravelPlanner 中，主要评价旅行计划是否符合常识、是否满足硬性约束以及执行步骤数。总体上，论文同时关注两方面：一是**效果**，即成功率或任务得分；二是**效率**，即步骤数和 Token 消耗。

**4.MemP 的整体工作流程是什么？**

首先，智能体从历史任务轨迹中提取具体步骤和抽象操作规则，形成程序性记忆；其次，面对新任务时，通过向量相似度找到相关记忆；然后，将检索结果加入模型上下文，指导智能体执行；最后，根据任务成功或失败的反馈新增、修正或删除记忆。
