# How Far Can LLMs Improve from Experience? Measuring Test-Time Learning Ability in LLMs with Human Comparison

大型语言模型能否像人一样，在不更新模型参数的情况下，仅通过测试阶段的实际经验，逐步学会更好的解题策略？

**EMNLP 2025**

## 一、研究背景

传统的大模型评测通常使用数学题、知识问答、代码题等静态数据集，主要衡量模型已经掌握了多少知识，以及它当前能把任务完成到什么程度。

但作者认为，智能不仅体现在“已经知道什么”，也体现在：

- 遇到陌生任务后，能否快速总结规律；
- 能否根据成功或失败的经验调整策略；
- 随着经验增加，表现能否持续提升。

现有评测大多只比较模型最后的成绩，并不限制模型训练期间看过多少相关数据，因此很难判断模型是在真正学习，还是仅仅依赖预训练记忆。

### 什么是测试时学习

整个过程通常**不更新模型参数**，而是把经验或策略放进上下文中，让模型在当前会话中学习。作者强调，理想的模型不仅应在少量经验后改善，而且这种改善还应随着经验积累而持续。

作者认为，很多静态任务容易出现两个问题：

- 模型可能已经在训练数据中见过相似题目；
- 性能提升可能来自记忆和检索，而不是学会了新策略。

因此，论文主要使用动态的语义游戏作为评测环境。这类游戏具有以下特点：

- 每局情况不同，不容易被“刷满”；
- 需要发现隐藏规律；
- 需要制定策略，而不只是回忆答案；

作者认为，一个合适的测试时学习任务应具备三个条件：难度适中、存在可学习的规律、无法只靠记忆解决。

## 二、研究方法

研究的核心方法是比较：

> **同一个模型，在没有经验时的表现，与获得历史经验后的表现是否不同。**

论文并不修改模型权重，而是先让模型完成若干轮任务，并记录：

- 模型与环境的交互；
- 每一轮的结果或奖励；
- 模型对失败和成功的反思；
- 模型根据经验总结出的策略。

然后，再让模型使用这些策略完成新的测试任务。

| 设置   | 模型获得的信息                | 作用                 |
| ---- | ---------------------- | ------------------ |
| 无策略  | 只有任务规则                 | 基准组                |
| 规则策略 | 规则 + 模型只根据规则总结的策略      | 判断仅增加一段“策略提示”是否有帮助 |
| 经验策略 | 规则 + 根据历史交互、奖励和反思生成的策略 | 测量真正的测试时学习         |
| 人类策略 | 规则 + 人类编写的高质量策略        | 估计模型在正确指导下的潜在上限    |

如果经验策略显著提高性能，就说明模型确实利用了过去经验，而不是仅仅因为提示变长。

**历史经验表示**

方案A：直接提供完整历史记录

把前几轮所有对话、奖励和反思都放进模型上下文。

优点是信息完整，但缺点是上下文非常长：

- 二十问任务约为 5,000 tokens；
- 谁是卧底任务约为 12,000 tokens。

方案B：把历史经验压缩成策略

让模型阅读历史交互，然后总结一份较短的策略。

策略长度平均只有约：

- 二十问：243 tokens；
- 谁是卧底：261 tokens。

实验发现，完整历史不仅成本更高，表现还不如压缩后的策略。因此，后续实验主要采用经验策略表示。

---

### 工作流程

任务交互 → 获得反馈 → 自我反思 → 总结策略 → 应用策略 → 更新策略。

- **模型执行任务**  
  模型先完成若干轮“二十问”或“谁是卧底”等任务。
- **记录经验**  
  保存每轮的交互过程、最终奖励或输赢结果，以及模型对成功和失败的反思。
- **生成策略**  
  模型根据这些历史经验，总结出一份简短的可执行策略。
- **带着策略重新测试**  
  将总结出的策略加入模型上下文，让模型完成新的任务，并与“没有历史经验”的模型比较。
- **持续更新策略**  
  在累积实验中，模型每完成一轮都会加入新经验并更新策略，连续测试50轮，观察性能是否稳定提高。

## 三、实验设计

### 实验设置

**任务**

AIME 2025 数学题：一个单轮数学推理任务，使用2025年的美国数学邀请赛题目

二十问：单智能体、多轮任务。模型需要从固定的157个候选词中猜出一个目标词，最多可以提出20个只能回答“是”或“否”的问题。任务成绩使用 **NDCG@20** 计算，正确答案被模型排得越靠前，得分越高。

谁是卧底：多智能体、多轮任务。评价指标是胜率。

**模型**

主要实验评估了三种当时较强的通用模型：

- GPT-4o
- Claude 3.5 Sonnet
- DeepSeek-V3

此外，论文还单独测试了若干“推理模型”，包括：

- OpenAI o1
- DeepSeek-R1
- Claude 3.7 Sonnet
- Gemini 2.5 Pro
- Qwen3 235B A22B

**固定经验实验**

固定经验实验的流程如下：

1. 模型先完成5轮任务；
2. 每轮保存交互、奖励和自我反思；
3. 模型根据5轮经验生成一份策略；
4. 使用该策略完成32个新测试样本；
5. 与没有经验的基准表现比较。

**累积经验实验**

固定经验实验只能说明“少量经验有没有用”，却不能说明模型能否持续学习。

因此，作者又设计了50轮累积实验：

- 无经验模型每轮独立完成任务；
- 有经验模型每轮完成后总结经验并更新策略；
- 比较两者随轮次变化的累计平均奖励；
- 每种条件重复采样3次，以减少随机波动。

**人类对照实验**

为了判断大模型的学习速度究竟如何，作者还招募了8名人类参与者，让他们完成相同的二十问任务。

每人连续完成20轮，并与表现最好的模型 Claude 进行比较。

目的是：不仅看模型“有没有变好”，还看它“学得是否像人一样快”。

### 实验结果

在两个游戏任务中，经验策略普遍优于无策略基准。

二十问任务

经验策略相对无策略的提升为：

- GPT-4o：约 +5.80%
- Claude 3.5 Sonnet：约 +6.33%
- DeepSeek-V3：约 +3.97%

谁是卧底任务

提升更明显：

- GPT-4o：约 +10.02%
- Claude 3.5 Sonnet：约 +25.04%
- DeepSeek-V3：约 +12.50%

**数学任务没有表现出明显的测试时学习**

在AIME 2025中：

- GPT-4o使用经验策略后成绩下降；
- Claude基本不变；
- DeepSeek-V3也有所下降。

这说明，经验能否被利用，与任务结构密切相关。

**经验增加后，模型并不一定持续进步**

在50轮二十问累积实验中：

- Claude总体能从累计经验中获益；
- GPT-4o提升较小，前期与基准几乎重合；
- DeepSeek-V3在积累约5轮经验后反而下降。

DeepSeek-V3则可能在策略更新中不断引入噪声或错误，使错误逐渐累积。

**人类学习速度明显快于模型**

在人类实验中，多数参与者只经过很少几轮，就能快速掌握二十问的核心策略。

20轮后：所有人类参与者的累计表现都超过了Claude；

**推理模型也没有自然表现出更强的测试时学习**

实验发现，o1、DeepSeek-R1等推理模型在加入模型自己总结的策略后，表现没有提高，部分模型甚至下降。

**模型为什么学得不稳定**

作者分析了性能突然下降的轮次。

大模型的失败主要分为两类：

**1.过度利用**

模型过于相信已有策略，执着于某个错误方向，不愿及时调整。

例如，模型一旦猜测目标是运动用品，就不断围绕这一类别提问，即使已有回答已经不支持该方向。

**2. 过度探索**

模型提出过于分散、无关或低信息量的问题，没有围绕当前候选范围逐步收缩。

这说明模型难以稳定平衡强化学习中的经典问题：

> **探索未知策略**与**利用已有策略**之间的平衡。

## 四、总结

### 创新点

**把评测重点从“模型会多少”转向“模型能否从经验中变强”：**以往大模型评测主要看模型在固定数据集上的最终成绩，难以区分模型是在真正推理，还是依赖预训练记忆。

**提出一套可控的测试时学习评测框架**

论文设计了两种互补的实验设置：

- **固定经验量**：先给模型5轮经验，再测试是否提升；
- **累积经验**：连续更新策略，观察50轮内能否稳定进步。

同时设置了无策略、规则策略、经验策略和人类策略等条件，从而区分性能提升究竟来自经验、额外提示，还是高质量策略本身。

### 不足

**任务范围仍然较窄**

实验主要集中在：

- 一个数学基准；
- 两个语义游戏。

这些任务可以反映部分策略学习能力，但不能代表真实世界中的长期规划、工具使用、科学探索或复杂决策。作者也承认，需要更广泛、更复杂的环境来验证结论。

### Q&A

**1.它和In-context Learning有什么区别？**

普通的上下文学习通常是给模型一些人工准备的示例，让模型模仿示例完成新任务。本文的经验来自模型自己与环境的真实互动，包括成功、失败、奖励和反思。模型需要主动从这些经验中发现规律，而不只是模仿输入示例。因此，测试时学习更强调**自主经验获取、策略总结和持续更新**。

**2.论文怎么证明提升确实来自经验，而不是因为提示词变长了？**

作者设置了多个对照条件。无策略组只有任务规则；规则策略组加入模型仅根据规则生成的策略；经验策略组加入根据历史交互、奖励和反思生成的策略；此外还有人类策略组。如果仅仅增加一段策略文本就能提高成绩，那么规则策略也应该有效。但实验中规则策略经常使表现下降，而经验策略普遍带来改善，因此作者认为经验本身发挥了作用。
