# Contextual Experience Replay for Self-Improvement of Language Agents

**ACL 2025**

让智能体在**不更新模型参数、不重新训练**的情况下，从过去执行任务的经历中总结经验，并在后续任务中复用这些经验。

## 一、研究背景

近年来，大语言模型不仅可以生成文本，还可以作为智能体与外部环境交互。

这类任务通常属于**序列决策问题**：智能体需要连续观察网页、思考下一步动作、执行动作，再根据新页面继续决策。

但论文指出，即使使用 GPT-4o 等较强模型，智能体在 WebArena 这类真实、复杂的网页环境中，成功率仍明显低于人类。

其中一个重要原因是：

> 智能体不了解具体网站的结构和操作规律，每遇到一个新任务，往往都要从头探索。

### 现有方法存在不足

此前已有一些方法尝试让智能体利用历史轨迹，但存在几个问题：

**直接保存完整轨迹**

一些方法会直接保存过去的“观察—动作”序列。

问题是：

- 轨迹通常很长；
- 包含大量无关细节；
- 上下文窗口成本高；
- 很难迁移到不同但相似的任务。

**依赖成功标签或真实奖励**

部分方法必须知道某条轨迹是否成功，才能判断经验是否值得保存。

但在真实应用中，智能体往往无法方便地获得准确的成功标签。

**记忆缺乏检索和动态更新机制**

有些方法只是把完整工作流程写入记忆，却没有针对当前任务检索最相关内容，随着经验增加，记忆会越来越冗余。

因此，作者希望设计一个系统，同时满足：

1. 能从过去轨迹中提炼通用经验；
2. 不需要训练模型参数；
3. 能在推理阶段持续学习；
4. 能根据当前任务检索相关经验；
5. 能适应复杂、真实的网页环境。

---

## 二、研究方法

CER 主要包括四个部分：

| 模块      | 作用             |
| ------- | -------------- |
| 经验蒸馏模块  | 从历史任务轨迹中提取通用经验 |
| 动态经验缓冲区 | 保存和更新经验        |
| 经验检索模块  | 根据当前任务选出最有用的经验 |
| 基础智能体   | 利用经验完成实际任务     |

### 1. 从轨迹中蒸馏经验

CER 不直接存储完整轨迹，而是从轨迹中提取两类内容：

**环境动态信息（Dynamics）**

这里的“动态”并不主要指页面动画，而是指智能体对环境结构和状态的认识，例如：

- 某个页面的主要内容；
- 页面可以用来做什么；
- 页面对应的网址；
- 从某页面能够进入哪些功能。

例如：

> 论坛列表页会展示不同论坛，可以从这里进入目标版块；其 URL 是某个固定地址。

这类经验帮助智能体更快地理解网站结构，甚至直接跳转到正确页面。

**操作技能（Skills）**

技能是较通用的决策模式或操作流程，例如：

> 如何发布一篇标题为 `{title}` 的帖子。

技能会包含分步骤说明：

1. 点击发布按钮；
2. 找到标题输入框；
3. 填写标题；
4. 填写正文；
5. 提交。

作者要求模型尽量将技能写得抽象、通用，而不是只适用于某一个具体任务。

例如，经验不会写成：

> 进入 books 论坛。

而会写成：

> 进入名称为 `{forum name}` 的论坛。

这样，该技能就能迁移到游戏、科技、图书等不同论坛版块。

### 2. 动态经验缓冲区

蒸馏出的环境动态和技能会被存入经验缓冲区。

缓冲区具有两个重要特点：

**持续积累**

智能体执行的任务越多，经验越丰富。

**合并和去重**

蒸馏模块在生成新经验时，也会看到已有经验，从而避免反复保存相同内容。

### 3. 根据当前任务检索经验

当智能体面对一个新任务时，CER 不会把全部经验都放进上下文。

它会把以下信息交给检索模块：

- 当前任务目标；
- 网站描述；
- 缓冲区中的全部技能或动态信息。

然后由模型选出最相关的 top-k 条经验。

论文实验中默认最多检索：

- 5 条环境动态信息；
- 5 条操作技能。

这样可以减少无关信息，控制上下文长度。

### 4. 在上下文中回放经验

检索出的经验会被转换成自然语言描述，加入智能体原本的提示词中。

普通智能体看到的内容可能只有：

- 当前任务；
- 当前网页观察；
- 可执行动作。

加入 CER 后，智能体还会看到：

- 网站结构知识；
- 相关操作技能；
- 过去总结出的有效决策模式。

模型参数没有改变，但由于上下文中增加了针对当前环境的经验，智能体的决策会发生改善。

---

### 工作流程

- **收集任务轨迹**  
    记录智能体完成网页任务时的观察、思考和操作过程。轨迹可以来自离线人工示范，也可以来自智能体在线执行任务的历史记录。
- **提炼经验**  
    从轨迹中总结两类信息：
  - **环境动态**：页面功能、网站结构、URL 等；
  - **操作技能**：完成某类任务的通用步骤。
- **更新经验库**  
    将新经验与已有经验合并、去重，存入动态记忆缓冲区。
- **检索相关经验**  
    面对新任务时，根据任务目标，从经验库中选出最相关的页面知识和操作技能。
- **上下文回放并执行**  
    将检索到的经验加入大模型的提示上下文，帮助智能体选择更合理的动作。

## 三、实验设计

### 实验设置

**数据集**
WebArena：包含 812 个任务，以文本形式给出
VisualWebArena：包含 910 个任务
**模型**

- GPT-4o
- **ReAct** 作为推理和动作输出框架；

**Baseline**

- WebArena 原始方法；
- AutoEval；
- BrowserGym + GPT-4o；
- CER 离线模式；
- CER 在线模式；
- CER 混合模式；
- Tree Search 等搜索方法。

其中主要基线是：

> GPT-4o + BrowserGym + ReAct，不使用经验记忆。

### 实验结果

WebArena

| 方法            | 平均成功率     |
| ------------- | --------- |
| WebArena 原始方法 | 15.0%     |
| AutoEval      | 20.2%     |
| BrowserGym 基线 | 24.3%     |
| CER 离线        | 33.4%     |
| CER 在线        | 33.2%     |
| CER 混合        | **36.7%** |

**补充分析与消融实验**

**Token 成本**

CER 会增加上下文长度，因此作者分析了额外 token 成本。

| 方法       | 成功率   | 输入 Token 增幅 |
| -------- | ----- | ----------- |
| ReAct 基线 | 24.3% | 0           |
| CER 离线   | 33.4% | 5.8%        |
| CER 在线   | 33.2% | 11.2%       |
| CER 混合   | 36.7% | 17.3%       |

混合模式输入 token 增加约 17.3%，但成功率相对提升约 52.5%。

使用**跨模板成功率**进行评估：

| 方法  | 跨模板成功率 |
| --- | ------ |
| 基线  | 44.7%  |
| CER | 60.0%  |

**是否适用于开源模型**

作者还在 Llama-3.1-70B 上测试

| 方法       | 成功率   |
| -------- | ----- |
| ReAct 基线 | 17.3% |
| CER 混合   | 22.0% |

说明 CER 不只对 GPT-4o 有效。

与采样方法结合

作者还测试了 CER 能否与其他增强方法组合。

在 WebArena 论坛任务中：

| 方法           | 成功率       |
| ------------ | --------- |
| CER          | 37.7%     |
| 单独采样与重排序     | 43.1%     |
| CER + 采样与重排序 | **52.6%** |

这表明 CER 与搜索、采样等方法是互补的，而不是互相替代。

**是否必须只从成功轨迹学习**

默认情况下，CER 同时从成功和失败轨迹中总结经验，因为真实场景中可能没有可靠的奖励信号。

作者只使用真实评估器判断为成功的轨迹：

| 方法       | 平均成功率 |
| -------- | ----- |
| 从全部轨迹学习  | 31.4% |
| 只从成功轨迹学习 | 33.5% |

**环境动态和技能是否都必要**

作者分别移除两类经验：

| 设置     | 论坛任务成功率   |
| ------ | --------- |
| 完整 CER | **37.7%** |
| 去除技能   | 33.3%     |
| 去除环境动态 | 35.1%     |

## 四、总结

### 创新点

**1.可与其他智能体方法组合**

CER 与 ReAct、采样重排序、搜索类方法并不冲突。更像一个**通用记忆增强模块**，而不是必须替换现有智能体架构的方法。

**2.实现了训练-free 的持续自我改进**，部署成本较低

### 不足

1.经验质量高度依赖轨迹质量：CER 需要从目标明确、信息较丰富的轨迹中提炼经验。若轨迹来自随机探索或包含大量错误操作，生成的经验可能不准确，甚至误导后续决策。

2.在线学习存在冷启动问题

3.依赖强模型完成经验蒸馏和检索：蒸馏模块、检索模块和决策模块都由大模型完成

4.上下文和调用成本会持续增加：它需要额外调用模型进行经验蒸馏和检索，同时向决策模型加入经验文本。
